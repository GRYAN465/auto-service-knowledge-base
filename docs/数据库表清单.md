# 智能客服知识库系统 — 数据库表清单

> 配套《总体开发计划.md》§5、需求文档 §12。本文件把需求 §12 的表名细化为**一期建表范围 + 关键字段**，作为后端 Flyway `V1__init_schema.sql` 的依据。
> 二期表（智能问答 / 客服实时辅助）只列表名，留到二期再建。
>
> 最近更新：2026-06-15

---

## 0. 通用约定

- **库 / 字符集**：MySQL 8，`utf8mb4` / `utf8mb4_general_ci`，引擎 InnoDB。
- **主键**：`id BIGINT UNSIGNED AUTO_INCREMENT`。
- **公共字段（BaseEntity，除关系表/日志表外都带）**：

  | 字段 | 类型 | 说明 |
  |---|---|---|
  | `id` | BIGINT | 主键 |
  | `create_by` | BIGINT | 创建人 id |
  | `create_time` | DATETIME | 创建时间（MyBatis-Plus 自动填充） |
  | `update_by` | BIGINT | 更新人 id |
  | `update_time` | DATETIME | 更新时间（自动填充） |
  | `deleted` | TINYINT | 逻辑删除 0/1（`@TableLogic`） |

- **命名**：表/字段 snake_case；系统管理 `sys_`、知识 `kb_`、开放 `open_`、二期 `qa_`/`cc_`/`asr_`/`rt_`。
- **枚举**：统一用字符串常量（见 §4），存 VARCHAR，避免魔法数字。
- 下表「关键字段」只列业务列，公共字段不再重复。

## 1. 一期表 · 系统管理（`sys_*`）

### sys_org 机构/部门表（树）
| 字段 | 类型 | 说明 |
|---|---|---|
| parent_id | BIGINT | 上级机构，根为 0 |
| name | VARCHAR(100) | 机构名称 |
| code | VARCHAR(64) | 机构编码（唯一） |
| type | VARCHAR(20) | 类型：COMPANY/DEPT |
| sort | INT | 排序 |
| status | VARCHAR(16) | ENABLED/DISABLED |

### sys_user 人员账号表
| 字段 | 类型 | 说明 |
|---|---|---|
| org_id | BIGINT | 所属机构 |
| username | VARCHAR(64) | 登录名（唯一） |
| password | VARCHAR(100) | BCrypt 密文 |
| real_name | VARCHAR(64) | 姓名 |
| nickname | VARCHAR(64) | 昵称 |
| avatar | VARCHAR(255) | 头像 url |
| email | VARCHAR(128) | 邮箱 |
| phone | VARCHAR(32) | 手机号 |
| gender | VARCHAR(8) | M/F/U |
| status | VARCHAR(16) | ENABLED/DISABLED |
| expire_time | DATETIME | 账号有效期（登录策略） |
| last_login_time | DATETIME | 最近登录时间 |
| last_login_ip | VARCHAR(64) | 最近登录 IP |
| remark | VARCHAR(255) | 备注 |

### sys_role 角色表
| 字段 | 类型 | 说明 |
|---|---|---|
| name | VARCHAR(64) | 角色名 |
| code | VARCHAR(64) | 角色编码（唯一，如 ADMIN/AUDITOR） |
| data_scope | VARCHAR(16) | 数据权限范围（预留：ALL/DEPT/SELF） |
| sort | INT | 排序 |
| status | VARCHAR(16) | ENABLED/DISABLED |
| remark | VARCHAR(255) | 备注 |

### sys_permission 菜单/按钮权限表（树）
| 字段 | 类型 | 说明 |
|---|---|---|
| parent_id | BIGINT | 上级，根为 0 |
| name | VARCHAR(64) | 名称 |
| type | VARCHAR(16) | DIR/MENU/BUTTON |
| permission_code | VARCHAR(128) | 权限码（如 `knowledge:article:create`） |
| route | VARCHAR(128) | 客户端页面 name（MENU 用，对应 PageRouter） |
| icon | VARCHAR(64) | 图标 |
| sort | INT | 排序 |
| status | VARCHAR(16) | ENABLED/DISABLED |

### sys_user_role 用户-角色（关系表，无公共字段）
`id, user_id, role_id`，唯一索引 `(user_id, role_id)`。

### sys_role_permission 角色-权限（关系表）
`id, role_id, permission_id`，唯一索引 `(role_id, permission_id)`。

### sys_login_log 登录日志
`id, user_id, username, login_ip, login_location, client, status(SUCCESS/FAIL), msg, login_time`。

### sys_operation_log 操作日志
`id, user_id, username, module, operation, method, request_uri, request_param(TEXT), status, error_msg(TEXT), cost_time, operation_time`。

### sys_api_log 开放 API 调用日志
`id, app_id, app_key, api_path, http_method, request_param(TEXT), response_code, client_ip, cost_time, create_time`。

## 2. 一期表 · 知识管理 / 应用（`kb_*`）

### kb_category 知识分类表（树，多级目录）
| 字段 | 类型 | 说明 |
|---|---|---|
| parent_id | BIGINT | 上级分类，根为 0 |
| name | VARCHAR(100) | 分类名 |
| code | VARCHAR(64) | 编码 |
| sort | INT | 排序 |
| status | VARCHAR(16) | ENABLED/DISABLED |

### kb_tag 知识标签表
`id + name VARCHAR(64)（唯一）, color VARCHAR(16), sort` + 公共字段。

### kb_article 知识主表
| 字段 | 类型 | 说明 |
|---|---|---|
| title | VARCHAR(200) | 标题 |
| category_id | BIGINT | 分类 |
| knowledge_type | VARCHAR(32) | 话术/培训/产品/办公文档（SCRIPT/TRAIN/PRODUCT/OFFICE） |
| summary | VARCHAR(500) | 摘要 |
| content | LONGTEXT | 富文本正文（HTML） |
| status | VARCHAR(20) | DRAFT/PENDING_AUDIT/ONLINE/OFFLINE/REJECTED（见 §4） |
| source | VARCHAR(16) | MANUAL/IMPORT |
| author_id | BIGINT | 作者 |
| current_version | INT | 当前版本号 |
| online_time | DATETIME | 上线时间 |
| offline_time | DATETIME | 下线时间 |
| offline_reason | VARCHAR(255) | 下线原因 |
| view_count | BIGINT | 浏览数（冗余计数） |
| like_count | BIGINT | 点赞数 |
| dislike_count | BIGINT | 点踩数 |
| favorite_count | BIGINT | 收藏数 |
| comment_count | BIGINT | 评论数 |

> 一期检索：对 `title`、`summary`、`content` 建 MySQL 全文索引或 LIKE 检索（不上 ES）。

### kb_article_tag 知识-标签（关系表）
`id, article_id, tag_id`，唯一索引 `(article_id, tag_id)`。

### kb_attachment 附件表
| 字段 | 类型 | 说明 |
|---|---|---|
| article_id | BIGINT | 所属知识 |
| file_name | VARCHAR(255) | 原始文件名 |
| file_path | VARCHAR(512) | 存储路径（FileStorage 抽象） |
| file_type | VARCHAR(32) | PDF/WORD/PPT/AUDIO/VIDEO/OTHER |
| file_size | BIGINT | 字节数 |
| storage_type | VARCHAR(16) | LOCAL（后续 MINIO） |
| download_count | BIGINT | 下载次数 |

### kb_article_version 知识版本表
`id, article_id, version_no, title, content(LONGTEXT 快照), editor_id, change_log, create_time`。

### kb_audit_record 知识审核记录表
`id, article_id, submit_by, auditor_id, audit_status(PASS/REJECT), audit_opinion, submit_time, audit_time`。

### kb_import_record 知识导入记录表
`id, file_name, total_count, success_count, fail_count, status(RUNNING/DONE/FAIL), fail_detail(TEXT), operator_id, create_time`。

### kb_download_log 知识下载日志表
`id, article_id, attachment_id, user_id, download_type(SINGLE/BATCH/ZIP), client_ip, create_time`。

### kb_favorite 知识收藏表
`id, article_id, user_id, folder VARCHAR(64), create_time`，唯一索引 `(article_id, user_id)`。

### kb_share 知识分享表
`id, article_id, from_user_id, to_user_id, share_type, message, read_status(0/1), create_time`。

### kb_comment 知识评论表
`id, article_id, user_id, parent_id(回复), content(TEXT), status(NORMAL/HIDDEN), like_count, create_time` + 逻辑删除。

### kb_like 点赞/点踩表
`id, target_type(ARTICLE/COMMENT), target_id, user_id, type(1 赞 / -1 踩), create_time`，唯一索引 `(target_type, target_id, user_id)`。

### kb_view_log 知识浏览记录表
`id, article_id, user_id, client_ip, stay_seconds, create_time`。

### kb_search_log 知识搜索日志表
`id, user_id, keyword, result_count, client_ip, create_time`。

### kb_usage_stat 知识使用统计表（按天聚合）
`id, article_id, stat_date, view_count, like_count, favorite_count, share_count, comment_count, download_count`，唯一索引 `(article_id, stat_date)`。

## 4. 关键状态枚举

| 字段 | 取值 |
|---|---|
| 通用 status | `ENABLED` / `DISABLED` |
| kb_article.status | `DRAFT`（草稿）→ `PENDING_AUDIT`（待审核）→ `ONLINE`（已上线）/ `REJECTED`（驳回）；上线后可 `OFFLINE`（已下线） |
| kb_audit_record.audit_status | `PASS` / `REJECT` |
| kb_article.knowledge_type | `SCRIPT`（话术）/ `TRAIN`（培训）/ `PRODUCT`（产品）/ `OFFICE`（办公文档） |
| kb_like.type | `1`（赞）/ `-1`（踩） |

## 6. Flyway 落地

- `V1__init_schema.sql`：建上述**一期全部表**（系统管理 + 知识 + 应用统计 + open_app）+ 索引。
- `V2__seed_rbac.sql`：种子数据 —— 超级管理员账号、内置角色（系统管理员/知识管理员/审核员/坐席/普通用户）、菜单与权限码（对应 §14.1 页面树 + 各模块按钮权限码），仅给 `ADMIN` 授全权。
- `V3__seed_module2_permissions.sql`：模块 2 系统管理按钮权限码补充并授 ADMIN。
- `V4__seed_module3_permissions.sql`：模块 3 分类/标签按钮权限码并授 ADMIN。
- `V5__seed_module4_permissions.sql`：模块 4 知识管理/附件按钮权限码并授 ADMIN。
- `V6__seed_module6_permissions.sql`：模块 6 补 `interaction:like` 权限码并授 ADMIN。
- `V7__grant_builtin_roles.sql`：修复 V2 种子缺口 —— 按职责幂等给 KNOWLEDGE_ADMIN/AUDITOR/AGENT/USER 四内置角色补授一期权限码（此前仅 ADMIN 有权，非 admin 账号一律 403）。
- `V8__grant_statistics.sql`：模块 7 数据统计授权 —— `statistics:view` 与菜单 401/目录 400 V2 已种且 ADMIN 已授，本迁移按 V7 遗留约定幂等补**非 admin 授权**：仅 KNOWLEDGE_ADMIN(2)/AUDITOR(3)（后台分析角色，含目录 400 作菜单父节点），坐席/普通用户不开放。仅授权、无新权限码、无新表。
- 一期各模块权限码以 `V3`–`V8` 增量补授；二期表与权限后续以更高版本号增量添加，**均不回改既有版本**。模块 8（开放 API）的非 admin 角色授权仍需在各自迁移中镜像 V7/V8 写法另行追加。

> **模块 7 数据统计为纯只读聚合，不新增表/实体**：复用 `kb_article`（计数列 + 状态/类型）、`kb_view_log`、`kb_search_log` 现有表。GROUP BY / SUM 由聚合 `StatisticsMapper`（不绑实体、不继承 BaseMapper，`@MapperScan` 扫描）的原生 `@Select` 承载，原生 SQL 查 `kb_article` 处手写 `deleted = 0` 以补偿 `@TableLogic` 不生效。`kb_usage_stat`（按天聚合）与 `kb_download_log` 仍仅有建表、尚无实体/Mapper，留待后续按天报表/下载分析时再接入。
