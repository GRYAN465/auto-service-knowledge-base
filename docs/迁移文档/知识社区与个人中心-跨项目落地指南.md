> 本文档从 **tuandui（智能客服知识库）** 已验收实现抽象而来，供架构相同、接口/权限码略有差异的 sibling 项目使用。

## 需求 1 — 个人中心页（personal.center）

### 功能
- 路由名：`personal.center`；菜单挂在「知识应用」下；**全部登录用户可见**（含管理员）
- 顶栏用户名点击也可进入个人中心
- 页面内容：
  1. **头像**：圆形、蓝底 `#2563EB`、白字、显示 `realName` 首字（无则 `username` 首字）
  2. **修改密码**按钮 → 对话框输入旧密码/新密码
  3. **SubTab 切换**（不要用嵌套 QTabWidget 造成双层白框）：
     - Tab A「发布的知识」：分页列表，本人用 `/knowledge/article/mine`，他人主页不用此页
     - Tab B「收藏的知识」：本人用 `/interaction/favorite`；**查看他人时也要显示此 Tab**
  4. 空状态文案：
     - 发布为空：「暂无发布的知识」/ 他人：「暂无已上线知识」
     - 收藏为空：「暂无收藏的知识」/ 他人：「对方暂无收藏」
     - 对方开启收藏隐私：「对方已将收藏设为隐私」
  5. **收藏隐私**（仅本人）：复选框「收藏仅自己可见」，调用 `PUT /user/profile/favorite-privacy`

### 组件拆分（参考）
| 组件 | 职责 |
|---|---|
| `ProfilePage` | 个人中心菜单页；`QStackedWidget`：主页 ↔ 详情 |
| `UserProfilePanel` | 可复用：`userId=0` 表当前用户；他人主页从详情作者进入 |
| `ChangePasswordDialog` | 改密 |
| `AvatarLabel` | 通用圆形头像 widget |

### 导航
- `ProfilePage`：卡片点击 → `ArticleDetailPanel` → 返回回主页
- `KnowledgeSearchPage` 内嵌 `UserProfilePanel`：返回 → 回到详情页（不是列表）

## 需求 2 — 知识检索页重构为「知识社区」

### 布局（自上而下）
1. **顶栏**：左「知识社区」标题；右 **模糊搜索框**（placeholder：模糊搜索标题/摘要/正文）
2. **排序切换**：按钮组「最新」「最热」
   - 最新：`sortBy=UPDATE_TIME`
   - 最热：`sortBy=VIEW_COUNT`
3. **常用标签区**（`SectionCard`）：
   - 左标题「常用标签」；右「编辑」按钮
   - 横向按钮：「全部」+ 用户 pin 的标签（来自 DB 全量标签的子集）
   - 点击标签筛选 `tagId`；未 pin 的标签不在此展示
4. **信息流**：`QScrollArea` + 卡片列表（非 QTableWidget）
5. **底部状态**：加载中 / 已加载 n/m / 暂无数据

### 无限滚动分页（重要，避免重复）
- **首屏**：`offset=0&pageSize=15`
- **后续每次**：`offset=已加载条数&pageSize=10`
- **禁止**首屏 page=1 pageSize=15 后再用 page=2 pageSize=10（会重复第 11–15 条）
- 监听 `QScrollBar::valueChanged`，距底部 ≤40px 触发 `loadMore`
- 防重入：`loading` 标志；筛选/排序/关键词变化时 `resetFeed()` 清空重载

### 页面栈（QStackedWidget）
- `listPage` | `ArticleDetailPanel` | `UserProfilePanel`
- 详情作者点击 → `openProfile(authorId)`

## 需求 3 — 信息流卡片（ArticleFeedCard）

每条展示：
- **标题**（加粗 16px）
- **正文预览** 20 字（优先 `contentPreview`，否则 `summary`，超出加「…」）
- **互动数据**：👍 点赞数 · 💬 评论数 · 👁 浏览数
- 整卡可点，`cursor: pointer`，hover 边框高亮
- 点击 emit `clicked(articleId)` → 打开详情

## 需求 4 — 常用标签

### 数据来源
- 标签全量来自后端：`GET /knowledge/tag/options`（读 `kb_tag`，按 sort 排序）
- **Pin 列表**存客户端本地：`QSettings`，key 按 `userId` 隔离（`PinnedTagsStore`）

### 交互
- 「编辑」→ 对话框多选标签 → 保存 pin 列表 → 刷新标签按钮行
- 首次无 pin 时：默认 pin 前 5 个标签
- pin 列表中已被 DB 删除的 tagId 自动过滤

## 需求 5 — 详情页作者区

- 详情 VO 增加：`authorId`、`authorName`、`authorUsername`
- 详情顶部展示：`AvatarLabel(36)` + 作者名（可点击）
- 点击 emit `authorClicked(authorId)` → 打开 `UserProfilePanel(authorId)`
- 他人主页 Tab：
  - 发布：`GET /search/article?authorId=&offset=&pageSize=&sortBy=`
  - 收藏：`GET /user/profile/{id}/favorites`（受隐私控制）

## 后端 API 清单（按参考契约实现，路径可按项目微调）

### 检索
```
GET /search/article
  ?keyword=&tagId=&authorId=&sortBy=UPDATE_TIME|VIEW_COUNT
  &offset=0&pageSize=15   # 信息流模式；传 offset 时忽略 page
响应 data: { total, page, pageSize, list: SearchArticleVO[] }
SearchArticleVO: id, title, summary, contentPreview, viewCount, likeCount, commentCount, tags[]
```

### 个人中心 / 资料
```
GET  /user/profile/{id}
     → { id, username, realName, onlineArticleCount, self, favoritePrivate?, favoritesVisible }
GET  /knowledge/article/mine?page=&pageSize=&status=
GET  /interaction/favorite?page=&pageSize=          # 本人收藏
GET  /user/profile/{id}/favorites?page=&pageSize=  # 他人公开收藏
PUT  /user/profile/favorite-privacy  { "favoritePrivate": true|false }
PUT  /auth/me/password               { "oldPassword", "newPassword" }
```

### 详情
```
GET /knowledge/article/{id}/view
    → 含 authorId, authorName, authorUsername, likeCount, commentCount, favorited...
```

### 标签
```
GET /knowledge/tag/options → [{ id, name, color, sort }]
```

### DB（Flyway）
- `sys_user.favorite_private TINYINT DEFAULT 0`（0 公开 1 隐私）
- 菜单 `sys_permission`: route=`personal.center`，授予全部内置角色 + ADMIN 全权限
- 演示数据：多标签带 color、知识带 contentPreview

### 权限码（示例，按项目映射）
| 能力 | 权限码 |
|---|---|
| 检索/社区 | `knowledge:search` |
| 收藏 | `interaction:favorite` |
| 改密 | 已登录即可 |
| 个人中心菜单 | 无 permission_code 或全员可见 |

### ADMIN 角色
- 代码层：`PermissionService` 对 `ADMIN` 返回全部 `sys_permission`
- DB 层：迁移脚本 `INSERT IGNORE` 同步 ADMIN 全部权限

## 4. Java 编码规范

### 分层
```
controller → service → mapper → entity
DTO: XxxVO / XxxRequest  与 Entity 分离
```

### 检索分页（offset 模式）
```java
// Controller
@RequestParam(required = false) Integer offset
if (offset != null) {
    return searchService.searchByOffset(offset, pageSize, ...);
}
return searchService.search(page, pageSize, ...);

// Service：offset 模式返回 PageResult，page = offset/pageSize + 1
```

### SearchArticleVO 字段
必须含 `contentPreview`（HTML  strip 后取前 20 字）、`likeCount`、`commentCount`、`viewCount`

### UserProfile
```java
UserProfileVO {
  Long id; String username; String realName;
  Long onlineArticleCount;
  Boolean self;              // 是否本人
  Boolean favoritePrivate;   // 仅 self=true 时返回
  Boolean favoritesVisible;  // 当前查看者能否看收藏
}
```

### 权限
- Controller：`@PreAuthorize("hasAuthority('knowledge:search')")`
- Swagger：`@Tag` / `@Operation(description = "见 API契约 §x")`
- ADMIN：`PermissionService.listGranted` 对 ADMIN 返回 `selectAllEnabled()`

## 7. 常见坑（务必写入 AI 上下文）

| 坑 | 正确做法 |
|---|---|
| 分页重复 | 统一 offset 模式，首屏 15、后续 10 |
| 双层白框 | SubTabButton + StackedWidget，勿嵌套 TabWidget |
| KnowledgeApi 不是 QObject | 用成员变量 `KnowledgeApi m_api`，勿 `new KnowledgeApi(parent)` |
| 他人主页无收藏 Tab | 始终显示 Tab，空/隐私用文案区分 |
| 标签乱码 | SQL 迁移用 utf8mb4，Windows 用 cmd 重定向导入 |
| ADMIN 缺菜单 | 代码层 ADMIN 全权限 + 迁移同步 role_permission |
| notify 命名空间 | 与项目现有 Notify / notify:: 保持一致 |
