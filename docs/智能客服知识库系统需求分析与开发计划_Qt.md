# 智能客服知识库系统需求分析与开发计划（Qt 技术栈）

> 本文档根据用户提供的需求图片（两张功能明细表 + 一张产品整体定位）整理，并结合 Qt 技术栈规划。  
> 系统是一个完整的智能客服知识平台，主线包含三块：知识中台（知识应用 / 管理 / 统计 / 开放）、智能问答（解析切分 / 向量化 / 大模型问答）、客服实时辅助（CC 集成 / ASR 实时转写 / 实时话术·流程图·脚本推送）。

---

## 1. 项目概述

### 1.1 系统名称

**智能客服知识库系统**

### 1.2 建设目标

建设一套面向客服、运营、知识管理员及外部业务系统的统一知识库平台，实现以下目标：

1. 对客服常用知识进行统一存储、维护、审核、上线、下线和分类管理。
2. 支持用户通过标题、知识内容、分类、标签等条件快速检索知识。
3. 支持知识详情查看、朗读、分享、收藏、评论、点赞、点踩等知识应用能力。
4. 支持知识录入、知识导入、知识审核、知识下载、分类标签管理等后台管理能力。
5. 支持机构人员、角色、菜单按钮权限、系统日志等基础系统管理能力。
6. 支持知识使用数据统计，展示热门知识、搜索行为和知识互动情况。
7. 支持通过 API 形式向外部应用开放知识检索、知识详情、知识问答能力。
8. 支持知识解析切分、向量化入库，并结合大模型实现智能知识问答。
9. 支持与 CC（呼叫中心）系统集成，呼叫接入后通过 ASR 将客户问题实时转写为文字。
10. 支持根据实时转写内容自动匹配话术、流程图、脚本并推送给客服坐席，辅助快速应答。

### 1.3 系统定位

本系统不是单纯的文档管理系统，而是一个完整的知识平台，覆盖：

```text
知识生产
  ↓
知识审核
  ↓
知识上线
  ↓
知识检索
  ↓
知识应用
  ↓
知识统计
  ↓
知识开放
  ↓
智能问答
  ↓
客服实时辅助（CC 接入 → ASR 转写 → 知识匹配 → 坐席实时推送）
```

系统核心价值是：

**实现知识统一管理、快速查找、便捷应用、数据沉淀、接口开放、智能问答与客服实时辅助。**

## 3. 需求范围说明

本系统不是单纯的文档管理系统，而是一个完整的智能客服知识平台。综合需求明细表与产品整体定位，系统主线由三块并行构成：

| 主线 | 说明 | 需求来源 |
|---|---|---|
| 知识中台 | 知识应用、知识管理、分类标签、审核上下线、统计、开放 API | 需求明细表 |
| 智能问答 | 知识解析切分、向量化入库、大模型语义问答 | 需求明细表 |
| 客服实时辅助 | 与 CC（呼叫中心）集成，呼叫接入后 ASR 实时转写，自动匹配并推送话术 / 流程图 / 脚本给坐席 | 产品整体定位 |

> 早期草案曾把“CC 对接 / ASR 转写 / 实时推荐”列为二期外部场景。鉴于产品整体定位明确要求本系统与 CC 系统集成、呼叫转写、向坐席实时推送话术（重点能力 ★★★★），本版已将其**纳入一期主线**，与知识中台、智能问答并列。详细需求见第 10 章（10.3 起）。

### 3.1 范围调整说明

## 5. 前端知识应用需求

### 5.1 知识检索

#### 需求说明

用户能够根据标题、知识内容等条件进行模糊检索，快速展示关联的多条检索结果。

#### 功能点

| 功能点 | 说明 | 优先级 |
|---|---|---|
| 标题检索 | 根据知识标题进行模糊匹配 | P0 |
| 内容检索 | 根据知识正文内容进行全文检索 | P0 |
| 分类筛选 | 根据知识分类进行过滤 | P0 |
| 标签筛选 | 根据知识标签进行过滤 | P0 |
| 多条件检索 | 支持标题、内容、分类、标签、时间等组合查询 | P1 |
| 搜索结果列表 | 展示标题、摘要、分类、标签、更新时间、浏览量 | P0 |
| 结果排序 | 支持按相关度、更新时间、热度排序 | P1 |
| 关键词高亮 | 对命中的关键词进行高亮显示 | P1 |
| 无结果提示 | 无搜索结果时提示用户调整关键词 | P0 |
| 无结果记录 | 记录无结果关键词，用于后续补充知识 | P1 |

#### Qt 实现建议

| 功能 | Qt 技术 |
|---|---|
| 搜索输入框 | QLineEdit / QML TextField |
| 搜索结果列表 | QTableView / QListView / QML ListView |
| 分类树 | QTreeView / QML TreeView |
| 标签筛选 | QComboBox / QListWidget |
| 网络请求 | QNetworkAccessManager |
| 本地缓存 | SQLite + QtSql |

### 6.5 知识审核

#### 需求说明

知识经过审核人员审核后，才可正常上线。

#### 知识状态

| 状态 | 说明 |
|---|---|
| 草稿 | 编辑人员保存但未提交 |
| 待审核 | 已提交，等待审核人员处理 |
| 审核通过 | 审核通过后可上线 |
| 审核驳回 | 审核不通过，退回修改 |
| 已上线 | 用户可检索、查看 |
| 已下线 | 用户不可检索或不可查看，按配置控制 |

#### 审核操作

| 操作 | 说明 | 优先级 |
|---|---|---|
| 提交审核 | 编辑人员提交 | P0 |
| 审核通过 | 审核人员确认 | P0 |
| 审核驳回 | 填写驳回原因 | P0 |
| 查看审核记录 | 查看每次审核意见 | P1 |
| 重新提交 | 修改后再次提交 | P0 |

## 10. 智能知识应用与客服实时辅助需求

> 本章包含两部分：基于知识库的智能问答（10.1～10.2），以及面向客服坐席的实时辅助能力（10.3～10.9，对应产品定位中“与 CC 集成、呼叫转写、实时推送话术/流程图/脚本”的核心要求）。

### 10.1 知识解析切分

#### 需求说明

结合知识管理和知识录入模块，将录入的知识进行解析、切分、向量化入库，构建知识问答库，供知识问答应用使用。

#### 处理流程

```text
知识录入 / 知识导入
  ↓
解析正文和附件
  ↓
清洗无效内容
  ↓
按段落、标题、语义进行切分
  ↓
生成知识文本块
  ↓
调用向量化模型生成向量
  ↓
写入向量库
  ↓
供知识问答检索使用
```

#### 功能点

| 功能点 | 说明 | 优先级 |
|---|---|---|
| 文本解析 | 解析富文本正文 | P1 |
| 附件解析 | 解析 PDF、Word、PPT、Excel 等文档 | P2 |
| 内容清洗 | 去除页眉页脚、乱码、空行、重复内容 | P1 |
| 知识切分 | 按标题、段落、长度、语义进行切分 | P1 |
| 文本块管理 | 记录文本块来源于哪条知识 | P1 |
| 向量化入库 | 调用 Embedding 模型生成向量 | P1 |
| 索引更新 | 知识修改、上线、下线时同步更新向量索引 | P1 |
| 解析状态 | 记录解析成功、失败、处理中 | P1 |
| 失败重试 | 支持重新解析和重新入库 | P1 |

#### 知识切分建议

| 切分方式 | 说明 |
|---|---|
| 按标题层级切分 | 适合制度、手册、说明文档 |
| 按段落切分 | 适合 FAQ、话术、流程说明 |
| 按长度切分 | 控制每个文本块长度，便于向量检索 |
| 保留上下文 | 每个文本块保留标题、分类、标签信息 |
| 重叠切分 | 相邻文本块保留一定重叠内容，提高问答准确率 |

### 10.2 知识问答

#### 需求说明

根据用户输入的问题，通过向量化语义检索，从知识向量库中匹配最符合的知识文本块，经过大模型总结后回复。

#### 处理流程

```text
用户输入问题
  ↓
问题向量化
  ↓
从向量库检索相似知识文本块
  ↓
按相似度、权限、状态过滤
  ↓
取 Top K 知识片段
  ↓
交给大模型总结
  ↓
返回答案 + 来源知识
```

#### 功能点

| 功能点 | 说明 | 优先级 |
|---|---|---|
| 问题输入 | 用户输入自然语言问题 | P1 |
| 语义检索 | 按语义匹配知识，而不只是关键词 | P1 |
| 权限过滤 | 用户只能问到自己有权限查看的知识 | P1 |
| 状态过滤 | 只使用已上线知识 | P1 |
| 大模型总结 | 将多个知识片段总结成自然语言答案 | P1 |
| 来源引用 | 答案中展示参考知识来源 | P1 |
| 追问支持 | 同一会话中支持连续提问 | P2 |
| 无答案处理 | 找不到答案时提示无匹配知识，并记录问题 | P1 |
| 问答日志 | 记录问题、命中文本块、回答、用户反馈 | P1 |

#### 问答展示示例

```text
问题：客户忘记密码怎么办？

回答：
客户忘记密码时，可引导客户进入登录页，点击“忘记密码”，
通过手机号验证码或邮箱验证完成密码重置。如客户无法验证身份，
需转人工核验。

参考知识：
1. 《用户密码重置流程》
2. 《账号安全校验规范》
```

### 10.3 客服实时辅助总体场景

#### 需求说明

```text

  ↓

  ↓

  ↓

  ↓
知识库语义匹配（复用问答向量检索能力）
  ↓
命中话术 / 流程图 / 脚本
  ↓
实时推送到坐席辅助面板
  ↓
坐席参考应答客户
  ↓

```

## 13. 核心业务流程

### 13.1 知识发布流程

```text
知识管理员录入知识
  ↓
保存草稿
  ↓
提交审核
  ↓
审核员审核
  ↓
审核通过
  ↓
知识上线
  ↓
生成全文检索索引
  ↓
知识解析切分
  ↓
向量化入库
  ↓
前端可检索、查看、问答调用
```

### 13.2 知识下线流程

```text
管理员选择知识
  ↓
点击下线
  ↓
填写下线原因
  ↓
知识状态变为已下线
  ↓
前端检索不再展示
  ↓
全文索引隐藏或删除
  ↓
向量索引同步禁用
```

### 13.3 知识问答流程

```text
用户输入问题
  ↓
系统识别问题文本
  ↓
问题向量化
  ↓
向量库检索相似知识块
  ↓
按权限、上线状态过滤
  ↓
调用大模型生成答案
  ↓
返回答案和参考知识
  ↓
记录问答日志和用户反馈
```

---

### 13.4 呼叫实时辅助流程

```text
客户来电接入 CC
  ↓
推送呼叫事件与音频流到知识平台
  ↓
ASR 实时转写为文字
  ↓
识别客户问题 / 意图
  ↓
语义检索匹配话术 / 流程图 / 脚本
  ↓
按权限与上线状态过滤
  ↓
实时推送推荐到坐席辅助面板
  ↓
坐席采纳并应答客户
  ↓
通话结束沉淀转写、推荐与采纳记录
```

## 19. 主要风险与应对

| 风险 | 表现 | 应对措施 |
|---|---|---|
| 知识质量不高 | 搜索和问答效果不好 | 制定知识模板、分类标签规范、审核标准 |
| 富文本兼容问题 | 编辑和展示效果不一致 | 前端统一富文本格式，优先使用 HTML 标准格式 |
| 附件解析复杂 | PDF、PPT、Word 内容抽取不稳定 | 一期先保证附件上传和下载，解析放到二期优化 |
| 权限复杂 | 菜单、按钮、数据权限容易混乱 | 先建立统一 RBAC 模型，再扩展数据权限 |
| 问答效果不稳定 | 大模型回答不准确 | 强制答案基于知识片段生成，并展示来源 |
| 系统范围膨胀 | 工期失控 | 明确 P0、P1、P2，优先完成知识管理和应用主链路 |
| 搜索性能不足 | 知识量大后检索变慢 | 使用 Elasticsearch/OpenSearch，建立合理索引 |
| 接口开放风险 | 外部系统越权访问 | 增加 AppKey、Secret、Token、限流和日志审计 |
| ASR 识别不准 | 转写错误导致推荐不准 | 引入行业热词与纠错词表，必要时做双轨/说话人分离 |
| CC 对接差异大 | 不同呼叫中心接口不一致 | 抽象 CC 适配层，先支持主用厂商再逐步扩展 |
| 实时性要求高 | 转写 / 推荐延迟影响坐席体验 | 流式处理 + WebSocket 推送，控制端到端延迟 |

## 20. 最终建议

按照图片中的需求，第一版应优先完成以下核心闭环：

```text
知识能录入
知识能审核
知识能上线
知识能检索
知识能查看
知识能互动
权限能控制
日志能审计
统计能展示
API 能开放
```

之后再逐步建设：

```text
知识解析切分
向量化入库
智能知识问答
大模型总结回答
客服实时辅助（CC 接入、ASR 转写、话术 / 流程图 / 脚本实时推送）
实时辅助效果优化（说话人分离、意图识别、多厂商适配）
```

这样更符合图片中的详细功能要求，也更容易用 Qt 技术栈稳定落地。
