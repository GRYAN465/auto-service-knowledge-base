## 2. 客户端架构（Qt Widgets，目录 `knowledgeAnswer/`）

### 2.1 目标目录结构

> 骨架已落地（2026-06-16）：`main.cpp` 入口 + `app/`（MainWindow / LoginDialog / PageRouter / Application / PlaceholderPage）+ `core/`（network / auth / config / logging / notify）+ `resources/`（`app.qss` 主题）；向导的 `mainwindow.{h,cpp,ui}` 已被 `app/MainWindow` 取代删除。`common/` 复用控件与 `modules/` 各业务真实页随对应模块开发再建（届时把 PageRouter 中对应 `name` 的占位页替换为真实页）。

```text
knowledgeAnswer/
├── CMakeLists.txt
├── main.cpp                 # 入口：QApplication → LoginDialog → MainWindow
├── app/                     # MainWindow(导航树+QStackedWidget)、LoginDialog、PageRouter、Application
├── core/                    # 跨模块基础设施
│   ├── network/             # ApiClient(QNetworkAccessManager 封装，注入 JWT)
│   ├── auth/                # Session(当前用户/token/权限码集合)
│   ├── config/              # Settings(QSettings 封装：baseUrl、记住账号)
│   ├── logging/             # Logger
│   └── notify/              # 全局提示 / 站内消息
├── common/                  # 复用控件：PagedTableView、SearchBar、CategoryTree、FormDialog、Pagination、RichTextViewer/Editor
├── modules/<module>/        # 每业务模块一目录：XxxPage(视图)+XxxController(逻辑)+XxxService(API)+Models
├── resources/               # qrc、图标、qss、i18n
└── tests/                   # Qt Test
```

### 2.2 模块内分层（所有模块统一套路）

| 类 | 职责 |
|---|---|
| `XxxPage : QWidget` | 纯视图：布局与控件；暴露信号，提供 `refresh()/setData()` |
| `XxxController` | 持有 Page + Service；处理交互、调 Service、回填视图、错误提示 |
| `XxxService` | 封装本模块 API 调用（基于 `core::ApiClient`），返回解析后的模型/错误 |
| `XxxTableModel : QAbstractTableModel` | 列表数据喂给 `QTableView`（分页、排序） |
| DTO 结构体 | 与后端 JSON 序列化/反序列化 |

### 2.3 核心基础设施与导航

- **ApiClient**（单例）：统一 baseUrl、注入 `Authorization: Bearer <jwt>`、解析统一响应 `{code,message,data}`、超时与错误信号、提供 `get/post/put/del`。
- **Session**：登录后保存当前用户、token、**权限码集合**；供导航过滤与按钮级权限判断。
- **导航与权限**：`MainWindow` 左侧导航树 + 中央 `QStackedWidget`；`PageRouter` 注册 `name→QWidget` 懒加载。登录后按权限码**过滤导航项**、隐藏无权页面；按钮级权限由各 Page 查询 `Session` 控制可见/可用。
- **视觉风格（基线）**：全局 QSS 主题 `resources/styles/app.qss`（单一蓝 `#2563EB` 强调色、白底卡片、8px 圆角、充裕留白、hover/selected 态、细化滚动条），`main.cpp` 用 `setStyleSheet` 注入。要求**高效简洁、落落大方、不毛坯**；新页面沿用既有 objectName 约定（`PrimaryButton` / `GhostButton` / `NavTree` / `TopBar` / `PageTitle` / `LoginCard` 等）让主题自动生效，样式优先扩展 app.qss。

## 3. 后端架构（Java / Spring Boot，目录 `server/`）

### 3.1 分包（单体应用，按功能分包）

```text
server/
├── pom.xml
└── src/main/
    ├── java/com/kb/
    │   ├── KbApplication.java
    │   ├── common/     # Result<T> 统一响应、全局异常、BaseEntity、分页、常量
    │   ├── security/   # Spring Security + JWT 过滤器、当前用户、权限码鉴权
    │   ├── config/     # Web/CORS/MyBatis-Plus/Swagger 配置
    │   ├── system/     # 系统管理: org/user/role/permission/log + auth(login/me)
    │   ├── knowledge/  # 知识: article/category/tag/attachment/version/audit/import/download
    │   ├── search/     # 检索（一期 MySQL）
    │   ├── interaction/# 收藏/分享/评论/点赞点踩/浏览
    │   ├── statistics/ # 统计
    │   ├── openapi/    # 开放API + AppKey/Secret + 限流 + 调用日志
    │   ├── ai/         # 调用 FastAPI 的客户端 + 问答编排（二期）
    │   └── realtime/   # CC/ASR/实时推荐 WebSocket（二期）
    └── resources/
        ├── application.yml        # MySQL/Redis/文件目录/FastAPI 地址
        ├── mapper/                # MyBatis XML（复杂查询）
        └── db/migration/          # Flyway：V1 建一期表，V2 种子 RBAC 数据
```

每模块标准分层：`controller → service → mapper(MyBatis-Plus) → entity`，DTO/VO 转换，统一 `Result<T>`，全局异常处理，分页用 MyBatis-Plus `IPage`。

### 3.2 安全与 RBAC

> 模块 1 已落地（2026-06-16）：`/auth/login` 已从 `sys_user` 查账号并用 BCrypt 校验密码，签发 JWT；`JwtAuthenticationFilter` 解析 token 后按用户加载权限码写入 Spring Security authorities；`/auth/me` 从 `sys_user/sys_role/sys_permission` 装配当前用户、角色、权限码与菜单树。禁用/过期账号不能登录，旧 token 也不会继续通过。

- `POST /auth/login` 校验后签发 JWT；`GET /auth/me` 返回用户 + **菜单树 + 权限码列表**（驱动客户端导航与按钮）。
- JWT 过滤器解析 token → 安全上下文；接口用权限码鉴权（`@PreAuthorize` 或自定义注解）。
- 数据权限（本部门/本分类）一期先留扩展点（先 RBAC，再扩数据权限）。

### 3.3 文件存储

本地目录 + 上传/下载接口；存储操作走接口抽象（`FileStorage`），便于后续换 MinIO。

### 3.4 设计模式落地

> 主骨架是「分层架构 + Spring IoC/AOP」；设计模式按需自然出现，不为用而用。下表为骨架阶段（2026-06-16 启动验证时）**已落地**或**预留**的模式，均可对应到代码（路径相对 `server/src/main/java/com/kb/`）。

| 模式 | 落地点 | 代码位置 |
|---|---|---|
| 分层架构 | `controller → service → mapper → entity`，配 DTO/VO 转换；模块 1 已按 `AuthController → AuthService/PermissionService → Sys*Mapper → Sys*Entity` 落地 | 全后端统一约定 |
| 门面 Facade | Controller 为各模块对外统一入口；`/auth/me` 把「用户 + 角色 + 权限码 + 菜单树」聚合成单个响应，客户端一次拿到构建导航所需的一切 | `system/auth/AuthController`、`system/auth/dto/MeResponse` |
| 责任链 Chain of Responsibility | Spring Security 过滤器链：JWT 过滤器用 `addFilterBefore(..., UsernamePasswordAuthenticationFilter.class)` 插入，逐段 `filterChain.doFilter` 下传 | `security/SecurityConfig`、`security/JwtAuthenticationFilter` |
| 模板方法 Template Method | `JwtAuthenticationFilter extends OncePerRequestFilter`（钩子 `doFilterInternal`）；`MyMetaObjectHandler` 的 `insertFill/updateFill` 由框架回调；MyBatis-Plus `IService/BaseMapper` 模板化 CRUD | `security/JwtAuthenticationFilter`、`config/MyMetaObjectHandler` |
| 策略 Strategy | `PasswordEncoder` → `BCryptPasswordEncoder`（可换）；**预留扩展点**：`FileStorage`（本地→MinIO）、`data_scope`（ALL/DEPT/SELF） | `security/SecurityConfig#passwordEncoder` |
| 静态工厂 Factory Method | `Result.ok()/ok(data)/fail()` 统一构造响应，避免到处 `new` | `common/Result` |
| 建造者 Builder | JWT `Jwts.builder()`；DTO/VO 用 Lombok `@Builder`（如 `MeResponse`） | `security/JwtTokenProvider`、各 VO |
| 代理 Proxy | MyBatis Mapper 接口运行期动态代理（无 impl 类）；`@EnableMethodSecurity` 下的 `@PreAuthorize`、`@Transactional` 走 AOP 代理 | `security/SecurityConfig`（`@EnableMethodSecurity`） |
| 单例 Singleton | 所有 `@Component/@Service/@RestController/@Configuration` Bean 默认单例，由容器托管 | 全局 |
| 依赖注入 / IoC | 构造器注入，如 `SecurityConfig(JwtTokenProvider)` | 全后端 |

补充约定（非 GoF）：统一响应 `Result<T>`（envelope）、DTO/VO 分离、逻辑删除（`deleted`）、公共字段自动填充（`MyMetaObjectHandler`）。

> 客户端（待建）沿用同构思路：`Page/Controller/Service/Model` 分层 + `ApiClient` 单例 + `PageRouter` 路由注册，详见 §2.2 / §2.3。

## 5. 前后端契约

- **REST**：HTTP + JSON，统一响应 `{ "code":0, "message":"ok", "data":... }`。
- **鉴权**：登录得 JWT，客户端存储并由 ApiClient 注入 `Authorization` 头。
- **分页**：请求 `{page,pageSize,...}`，响应 `{total,list}`。
- **实时（二期）**：WebSocket 用于分享/评论站内提醒、坐席辅助推送。
- **数据模型**：复用需求文档 §12 表（`sys_*`、`kb_*`、`qa_*`，二期 `cc_*`/`asr_*`/`rt_*`）。详见《数据库表清单.md》《API契约.md》。

## 8. 验证方式（端到端）

**脚手架阶段**
- 后端：`mvn spring-boot:run` → 访问 Swagger UI 与 `/actuator/health`；`POST /auth/login` 返回 JWT；`GET /auth/me` 带 token 返回用户；连 MySQL 确认 Flyway 已建表。
- 客户端：CMake 配置+构建+运行 → 登录框登录 → 进入主窗口 → 左侧导航完整、点击各项切换到对应空壳页。
- AI 服务：`uvicorn app.main:app` → `/health` 返回 200。

**每模块**：后端用 Swagger 自测 CRUD；客户端页面联调（增删改查、分页、按权限码控制的按钮）。

**一条竖切冒烟**（阶段一中段验证）：登录 → 新增分类 → 录入知识 → 提交审核 → 审核通过上线 → 前端检索命中 → 打开详情。
